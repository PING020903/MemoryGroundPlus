# 内存池管理系统测试用例设计

## 一、测试架构设计

### 1.1 测试分层
- **单元测试层**: 单个函数功能验证
- **集成测试层**: 多函数组合使用场景
- **压力测试层**: 极限条件和性能测试
- **异常测试层**: 错误输入和异常情况处理

### 2.2 测试框架建议
建议使用简单的测试框架结构：
```c
typedef struct {
    const char* name;
    void (*test_func)(void);
    int passed;
} test_case_t;
```

## 二、详细测试用例

### 2.1 基础功能测试 (Basic Functionality Tests)

#### Test 001: 内存池创建 - 正常情况
- **目的**: 验证基本内存池创建功能
- **前置条件**: 分配一块足够大的内存
- **测试步骤**:
  1. 准备 1KB 连续内存空间
  2. 调用 `mgp_create_with_pool()`
  3. 检查返回值非 NULL
- **预期结果**: 
  - 返回有效的内存池句柄
  - 初始分配计数为 0
- **调试配置**: `MGP_CREATE_POOL_DEBUG = 1`

#### Test 002: 内存池创建 - 最小尺寸
- **目的**: 验证最小可用内存池创建
- **测试步骤**:
  1. 计算最小尺寸：`sizeof(mgp_pool_t) + sizeof(mgp_ctrl_t)`
  2. 使用该尺寸创建内存池
- **预期结果**: 创建成功

#### Test 003: 内存池创建 - 尺寸不足
- **目的**: 验证错误处理
- **测试步骤**:
  1. 使用小于最小尺寸的内存创建池
- **预期结果**: 返回 NULL

#### Test 004: 最大可分配尺寸查询
- **目的**: 验证 `mgp_canAllocMaxSize()` 准确性
- **测试步骤**:
  1. 创建已知大小的内存池
  2. 调用查询函数
  3. 手动计算理论最大值并对比
- **预期结果**: 返回值与理论值一致

### 2.2 内存分配测试 (Memory Allocation Tests)

#### Test 010: 单次分配 - 小尺寸
- **目的**: 验证基本分配功能
- **测试步骤**:
  1. 创建 256 字节内存池
  2. 分配 10 字节
  3. 检查返回指针有效性
  4. 写入数据并验证
- **调试配置**: `MGP_MEMORY_ALLOC_DEBUG = 1, MGP_FREE_BLOCK_DEBUG = 1`
- **预期结果**: 
  - 返回有效指针
  - 可正常读写数据
  - 控制块正确初始化

#### Test 011: 单次分配 - 大尺寸
- **目的**: 验证大尺寸分配
- **测试步骤**:
  1. 分配接近最大可分配尺寸的内存块
- **预期结果**: 分配成功

#### Test 012: 多次连续分配
- **目的**: 验证多次分配的内存布局
- **测试步骤**:
  1. 依次分配 3 个不同大小的块 (32B, 64B, 128B)
  2. 检查各块地址不重叠
  3. 分别写入不同数据模式
  4. 验证头尾保护标记
- **预期结果**: 
  - 所有块地址不重叠
  - 数据写入正确
  - 头尾标记值为 `MGP_GUARD_HEAD/TAIL`

#### Test 013: 分配后写满整个池
- **目的**: 验证内存耗尽情况
- **测试步骤**:
  1. 循环分配直到失败
  2. 记录成功分配的次数
  3. 验证最后一次分配失败返回 NULL
- **预期结果**: 
  - 前面的分配都成功
  - 最后一次返回 NULL

### 2.3 内存释放测试 (Memory Free Tests)

#### Test 020: 单次释放
- **目的**: 验证基本释放功能
- **测试步骤**:
  1. 分配一块内存
  2. 调用 `mgp_free()` 释放
  3. 检查分配计数减 1
- **调试配置**: `MGP_MEMORY_FREE_DEBUG = 1`
- **预期结果**: 
  - 释放成功
  - 控制块计数正确更新

#### Test 021: 乱序释放
- **目的**: 验证任意顺序释放的正确性
- **测试步骤**:
  1. 分配 3 块内存：mem1, mem2, mem3
  2. 按 mem2 -> mem1 -> mem3 顺序释放
  3. 每次释放后检查剩余块数
- **预期结果**: 
  - 每次释放都成功
  - 控制块数组保持紧凑

#### Test 022: 重复释放同一块
- **目的**: 验证错误检测
- **测试步骤**:
  1. 分配一块内存
  2. 释放该块
  3. 再次尝试释放同一块
- **预期结果**: 
  - 第二次释放应打印错误信息
  - 不会导致系统崩溃

#### Test 023: 释放无效地址
- **目的**: 验证非法地址检测
- **测试步骤**:
  1. 创建一个栈变量指针
  2. 尝试释放该指针
- **预期结果**: 打印错误信息 "this pos is not alloc."

### 2.4 内存碎片整理测试 (Fragmentation Tests)

#### Test 030: 碎片产生与再利用
- **目的**: 验证碎片自动整理机制
- **测试步骤**:
  1. 分配 3 块：A(32B), B(64B), C(32B)
  2. 释放中间的 B
  3. 再分配一块 60B 的内存 D
  4. 检查 D 是否复用了 B 的空间
- **调试配置**: `MGP_FREE_BLOCK_DEBUG = 1`
- **预期结果**: 
  - D 的地址应该是之前 B 的地址
  - 证明碎片被有效利用

#### Test 031: 不同尺寸碎片的适配
- **目的**: 验证尺寸匹配策略
- **测试步骤**:
  1. 创建两个空隙：空隙 1(50B), 空隙 2(100B)
  2. 申请 80B 内存
  3. 检查分配到哪个空隙
- **预期结果**: 应该分配到空隙 2（空隙 1 太小）

#### Test 032: 完全填充测试
- **目的**: 验证无碎片时的分配
- **测试步骤**:
  1. 分配 N 块填满整个池
  2. 释放所有块
  3. 重新分配一大块内存
- **预期结果**: 应该能分配成功（空间被回收）

### 2.5 边界条件测试 (Boundary Condition Tests)

#### Test 040: 零长度分配
- **目的**: 验证边界输入处理
- **测试步骤**:
  1. 尝试分配 0 字节
- **预期结果**: 应返回 NULL 或有效指针（取决于设计决策）

#### Test 041: 极小尺寸分配
- **目的**: 验证最小可用分配
- **测试步骤**:
  1. 分配 1 字节
  2. 验证实际可用空间包含头尾标记
- **预期结果**: 分配成功，实际占用 `1 + 2*sizeof(unsigned int)`

#### Test 042: 最大尺寸分配
- **目的**: 验证极限分配
- **测试步骤**:
  1. 查询最大可分配尺寸 MAX
  2. 尝试分配 MAX 字节
- **预期结果**: 应该成功

#### Test 043: 超最大尺寸分配
- **目的**: 验证超限处理
- **测试步骤**:
  1. 查询最大可分配尺寸 MAX
  2. 尝试分配 MAX+1 字节
- **预期结果**: 返回 NULL

#### Test 044: 空指针释放
- **目的**: 验证 NULL 指针处理
- **测试步骤**:
  1. 调用 `mgp_free(pool, NULL)`
- **预期结果**: 触发 assert 或安全处理

#### Test 045: 空池分配
- **目的**: 验证 NULL 池句柄处理
- **测试步骤**:
  1. 调用 `mgp_malloc(NULL, 32)`
- **预期结果**: 触发 assert 或返回 NULL

### 2.6 数据完整性测试 (Data Integrity Tests)

#### Test 050: 头尾标记保护验证
- **目的**: 验证内存损坏检测机制
- **测试步骤**:
  1. 分配一块内存
  2. 故意修改头部标记值
  3. 尝试释放或进行其他操作
  4. （需要添加检测函数）
- **预期结果**: 应能检测到标记损坏

#### Test 051: 缓冲区溢出测试
- **目的**: 验证用户数据区保护
- **测试步骤**:
  1. 分配 32 字节
  2. 写入 40 字节数据（故意溢出）
  3. 检查是否覆盖尾部标记
- **预期结果**: 尾部标记被破坏（说明保护机制必要）

#### Test 052: 多块数据独立性
- **目的**: 验证块间隔离
- **测试步骤**:
  1. 分配 3 块相邻内存
  2. 分别写入不同模式数据（0xAA, 0x55, 0xFF）
  3. 互相验证没有串扰
- **预期结果**: 每块数据保持独立

### 2.7 压力测试 (Stress Tests)

#### Test 060: 大量小块分配
- **目的**: 验证高频分配性能
- **测试步骤**:
  1. 循环 100 次：分配->释放 8 字节小块
  2. 统计总耗时
  3. 检查最终状态正确性
- **预期结果**: 
  - 所有操作成功
  - 最终分配计数归零

#### Test 061: 随机尺寸混合操作
- **目的**: 验证复杂场景稳定性
- **测试步骤**:
  1. 使用伪随机数生成器
  2. 随机决定分配或释放
  3. 随机选择尺寸 (8~256 字节)
  4. 执行 1000 次操作
- **预期结果**: 
  - 不崩溃
  - 最终能释放所有块

#### Test 062: 内存池生命周期测试
- **目的**: 验证长期运行稳定性
- **测试步骤**:
  1. 创建内存池
  2. 重复执行：分配 10 块 -> 使用 -> 释放 5 块
  3. 循环 100 轮
- **预期结果**: 
  - 每轮操作都成功
  - 无内存泄漏

### 2.8 控制块排序测试 (Control Block Sorting Tests)

#### Test 070: 控制块自动排序验证
- **目的**: 验证 qsort 排序正确性
- **测试步骤**:
  1. 以不同顺序分配 5 块内存
  2. 分配后会触发内部排序
  3. （需添加调试接口查看排序结果）
- **预期结果**: 控制块按地址从高到低排列

#### Test 071: 排序模式切换测试
- **目的**: 验证不同排序模式
- **测试步骤**:
  1. 修改 `CTRL_BLOCK_QSORT_MDOE` 为 `LOW_TO_HIGH`
  2. 执行分配操作
  3. 验证排序方向改变
- **预期结果**: 控制块按地址从低到高排列

## 三、测试辅助工具

### 3.1 测试宏配置
```c
// 测试时启用所有调试输出
#define MGP_DEBUG 1
#define MGP_CREATE_POOL_DEBUG 1
#define MGP_CTRL_BLOCK_DEBUG 1
#define MGP_FREE_BLOCK_DEBUG 1
#define MGP_MEMORY_FREE_DEBUG 1
#define MGP_MEMORY_ALLOC_DEBUG 1
```

### 3.2 测试辅助函数
```c
// 1. 内存池状态检查
void mgp_dump_pool_info(mgp_t pool);
// 打印：总大小、已用大小、块数量、每个块的详细信息

// 2. 控制块验证
int mgp_validate_ctrl_block(mgp_ctrl_t *ctrl);
// 检查头尾标记是否正确

// 3. 内存池完整性检查
int mgp_check_pool_integrity(mgp_t pool);
// 检查所有控制块无冲突、无越界

// 4. 填充测试模式
void fill_pattern(void *buf, size_t size, uint8_t pattern);
// 用特定模式填充缓冲区便于观察

// 5. 比较测试模式
int check_pattern(void *buf, size_t size, uint8_t expected);
// 检查缓冲区内容是否符合预期
```

### 3.3 测试结果统计
```c
typedef struct {
    int total_tests;
    int passed_tests;
    int failed_tests;
    const char* failed_test_names[100];
} test_summary_t;
```

## 四、测试执行流程

### 4.1 分级执行策略
1. **Level 1 - 冒烟测试**: Test 001, 010, 020, 030
2. **Level 2 - 功能测试**: 所有基础功能测试 + 边界测试
3. **Level 3 - 完整测试**: 全部测试用例
4. **Level 4 - 压力测试**: 所有压力测试用例

### 4.2 测试报告格式
```
=====================================
Test Case: [名称]
Status: PASS / FAIL
Details: [详细信息]
=====================================

测试总结:
总计：XX 项
通过：XX 项
失败：XX 项
通过率：XX%
```

## 五、优先级建议

### P0 - 必须测试（核心功能）
- Test 001: 基本创建
- Test 010: 基本分配
- Test 020: 基本释放
- Test 012: 多次分配
- Test 021: 乱序释放

### P1 - 重要测试（边界和异常）
- Test 003: 创建失败处理
- Test 022: 重复释放
- Test 023: 无效地址释放
- Test 043: 超限分配
- Test 030: 碎片整理

### P2 - 完善测试（性能和极端情况）
- 所有压力测试
- 数据完整性测试
- 控制块排序测试

## 六、自动化测试建议

### 6.1 测试运行器
```c
#define RUN_TEST(test_func) do { \
    printf("Running %s...\n", #test_func); \
    test_func(); \
    if(test_passed) summary.passed_tests++; \
    else { summary.failed_tests++; summary.failed_test_names[summary.failed_tests] = #test_func; } \
    summary.total_tests++; \
} while(0)
```

### 6.2 CI/CD 集成
- 每次编译后自动运行 Level 1 测试
- 每日构建运行 Level 3 完整测试
- 版本发布前运行 Level 4 压力测试

---

**备注**: 
1. 部分测试需要添加额外的调试接口函数（如 `mgp_showAllocBlock` 的增强版）
2. 建议在测试文件中保留详细的注释说明测试意图
3. 对于失败的测试，应提供足够的诊断信息
